# 8_cg

컴퓨터 그래픽스 팀프로젝트: 3D 미니 전시관  
주제: 갈릴레이 오비트 뮤지엄

## 1. 프로젝트 소개

본 프로젝트는 OpenGL 기반의 C++ 3D 전시관 프로그램이다.  
전시관은 목성과 갈릴레이 위성을 주제로 구성하였으며, 사용자는 1인칭 카메라를 통해 전시관 내부를 이동하며 목성계 전시물을 관람할 수 있다.

전시관은 크게 사각형 갤러리, 계단 구조, 원형 플라네타륨 돔, 그리고 돔 중앙에 배치된 목성계 전시물로 구성된다.  
목성과 4대 위성인 이오, 유로파, 가니메데, 칼리스토는 텍스처가 적용된 구체로 표현하였고, 자전 및 공전 애니메이션을 통해 천체의 움직임을 시각적으로 나타냈다.

## 2. 개발 환경

- C++
- OpenGL
- GLFW
- GLAD
- GLM
- Visual Studio
- Windows 환경 기준

## 3. 필요한 라이브러리 및 리소스

본 프로젝트 실행을 위해 다음 라이브러리와 리소스가 필요하다.

### 라이브러리

- GLFW
- GLAD
- GLM

### 리소스 파일

- `shaders/` 폴더의 vertex shader, fragment shader 파일
- `textures/` 또는 `resources/` 폴더의 텍스처 이미지 파일
- 안내판 이미지 및 행성 텍스처 파일

실행 시 shader 파일과 texture 파일의 상대 경로가 유지되어야 한다.  
따라서 실행 파일만 따로 옮기지 말고, 프로젝트 폴더 구조를 유지한 상태에서 실행해야 한다.

## 4. 실행 방법

## 조작법

### 카메라 이동
| 입력 | 동작 |
|------|------|
| `W` | 앞으로 |
| `S` | 뒤로 |
| `A` | 왼쪽 |
| `D` | 오른쪽 |
| `Space` (이동 중) | 질주 (빠르게 이동) |

### 시점 변경
| 입력 | 동작 |
|------|------|
| `마우스 우클릭 + 드래그` | 시점 회전 (Yaw / Pitch) |
| `마우스 스크롤` | 줌 (FOV 1° ~ 45°) |

### 종료
| 입력 | 동작 |
|------|------|
| `ESC` | 프로그램 종료 |

### 실행 시 주의사항

- shader 파일 경로가 맞지 않으면 화면이 제대로 출력되지 않을 수 있다.
- texture 파일 경로가 맞지 않으면 행성 또는 안내판 이미지가 적용되지 않을 수 있다.
- OpenGL, GLFW, GLAD 관련 라이브러리 연결이 누락되면 빌드 오류가 발생할 수 있다.
- 프로젝트 폴더 구조를 유지한 상태에서 실행하는 것을 권장한다.

## 5. 폴더 구조

```txt
8_cg/

├─ headers/          # 클래스 헤더 파일

├─ docs/             # 보고서, 회의록, 발표 자료
